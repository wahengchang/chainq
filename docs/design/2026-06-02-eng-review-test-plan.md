## Test Harness Enabler (build FIRST, step 1)
- G2 fake-model profile (echo / cat fixture) — lets the entire engine run offline and
  deterministically. Every engine test below depends on it. Without it, tests either hit
  real CLI models (slow, non-deterministic, burns quota) or can't run at all.

## Critical Paths (must work, highest priority)
- Merkle cache invalidation: edit upstream node A's prompt → A and ALL transitive
  downstream (B, C…) go stale and re-run; unrelated sibling branches stay `cached`.
  This is regression-class — if wrong, the tool serves stale output silently.
- Atomic write: subprocess killed / times out mid-write → no half-written output is ever
  recorded as a valid cache entry (temp file + rename).
- Pin isolation: a pinned-sample trial run writes ONLY to .chain/scratch/, never touches
  .chain/outputs/ or 成品 (write) files.
- The wedge (E2E): in the iteration pane, edit a prompt node → only that node spawns a
  subprocess → new output streams back; upstream nodes show `cached`, fire no subprocess.

## Key Interactions to Verify (iteration pane = Runtime 模式)
- Edit prompt → run this node → see streamed output (the core loop).
- "Run to here" on a node → upstream stale ai node silently re-runs (per D3); per-node
  status (ran / cached) is visible live; "called N ai nodes" summary shown.
- Re-run any single node using existing upstream history (no upstream re-run).
- Save-back → validate(flow) runs → invalid flow is refused, nothing written.
- Two `chainq ui`/`run` on same flow → second warns (single-writer lock), no silent race.

## Edge Cases
- from: points at nonexistent node / dependency cycle → caught by validate() pre-run,
  all errors collected at once (not fail-on-first), with "did you mean X" edit-distance hint.
- ai/cmd subprocess: nonzero exit, timeout (SIGTERM→SIGKILL), stderr passed verbatim,
  CLI auth/login expired mid-chain → distinct "login expired" error, NOT cached as success.
- which-check pre-run: profile's CLI binary missing → blocked before burning anything.
- Empty / single-node flow; node renamed (identity = YAML key → old cache orphaned, re-runs).

## Affected Pages/Routes (local UI)
- chainq ui <flow.yaml> — the iteration pane (node list + 3-column input/prompt/output editor).

## Out of v1 scope (no tests yet)
- Loops, C4 schema+retry, write/成品 nodes, F4 structural canvas editing, tier-2 JS sandbox.
